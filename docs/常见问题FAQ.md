# 常见问题解答 (FAQ)

## 📌 基础问题

### Q1: 这个系统能准确预测彩票吗？
**A:** 不能保证准确预测。彩票本质上是随机事件，本系统只是基于历史数据的统计分析，提供概率参考。预测结果仅供参考，请理性投注。

### Q2: 预测准确率有多高？
**A:** 基于历史回测：
- Top-1 准确率（预测第一名命中）：约 15-20%
- Top-3 准确率（真实值在前3名）：约 40-50%
- Top-5 准确率（真实值在前5名）：约 60-70%

相比之下，随机猜测的 Top-1 准确率只有 8.3%（1/12）。

### Q3: 为什么使用 XGBoost 而不是其他算法？
**A:** XGBoost 的优势：
- 处理表格数据效果好
- 支持特征重要性分析
- 训练速度快
- 不易过拟合
- 社区支持好

我们也尝试过 RandomForest、SVM 等，XGBoost 综合表现最好。

## 🔧 技术问题

### Q4: 运行脚本时出现 "ModuleNotFoundError" 怎么办？
**A:** 安装所需的依赖包：
```bash
pip install pandas numpy requests xgboost scikit-learn
```

### Q5: 如何更新到最新的历史数据？
**A:** 有两种方式：

**方式1：从接口自动获取**
```bash
cd utils
python3 update_and_check_data.py
```

**方式2：手动添加**
```bash
cd utils
python3 add_2026106.py -p 2026107 -s 22 -z 10 -n 5 12 23 34 41 48 -y
```

### Q6: 预测需要多长时间？
**A:** 
- 使用完整历史数据（predict_next.py）：约 30-60 秒
- 仅使用2026年数据（predict_2026_only.py）：约 10-20 秒

主要时间花在模型训练上。

### Q7: 可以只使用部分年份的数据吗？
**A:** 可以。修改 `predict_next.py` 中的参数：
```python
df = fetcher.fetch_history(start_year=2024, end_year=2026)
```

或者直接使用 `predict_2026_only.py`。

### Q8: 如何提高预测准确率？
**A:** 建议：
1. **使用更多历史数据** - 数据越多，模型越稳定
2. **定期更新模型** - 每期开奖后重新训练
3. **特征工程优化** - 根据实际需求调整特征
4. **超参数调优** - 使用网格搜索或贝叶斯优化
5. **集成更多模型** - 增加子模型数量

但请记住，彩票的随机性决定了准确率有上限。

## 📊 数据问题

### Q9: 数据来源是什么？
**A:** 数据来自澳门六合彩历史开奖记录接口：
```
https://history.macaumarksix.com/history/macaujc2/y/{year}
```

### Q10: 数据多久更新一次？
**A:** 理论上每天开奖后都会更新。建议每期开奖后运行更新脚本。

### Q11: 如果数据有误怎么办？
**A:** 
1. 检查备份文件（`data/*.backup_*`）
2. 从接口重新获取正确数据
3. 使用 `add_2026106.py` 手动修正

### Q12: 为什么要备份数据？
**A:** 防止：
- 误操作导致数据丢失
- 程序 bug 破坏数据
- 方便回滚到之前的状态

每次添加新数据时会自动备份。

## 🎯 使用问题

### Q13: 如何选择使用哪个预测脚本？
**A:** 
- **predict_next.py**: 推荐！使用多年数据，更准确
- **predict_2026_only.py**: 快速预测，关注近期趋势
- **predict_2025_2026.py**: 平衡选择

### Q14: 预测结果中的概率是什么意思？
**A:** 概率表示该生肖成为下一期特码的可能性。

例如：鼠 38.43% 表示模型认为鼠有 38.43% 的概率成为特码。

所有生肖的概率之和为 100%。

### Q15: 应该相信高概率还是低概率？
**A:** 这取决于你的策略：
- **保守策略**: 选择高概率生肖（如前3名）
- **冒险策略**: 选择低概率生肖（博冷门）
- **平衡策略**: 结合概率和遗漏值

没有绝对正确的答案。

### Q16: 为什么同一期多次运行结果不同？
**A:** 可能原因：
1. 数据已更新（新增了一期）
2. 模型训练的随机性（虽然设置了种子）
3. 后处理的微小差异

正常情况下，结果应该非常接近。

## 🛠️ 开发问题

### Q17: 如何添加新的特征？
**A:** 
1. 在 `FeatureEngineer` 类中添加特征计算方法
2. 在 `_build_single_feature` 中调用新方法
3. 确保特征名称唯一
4. 测试新特征的效果

示例：
```python
def _build_single_feature(self, current_idx):
    feature = {}
    # ... 现有特征 ...
    
    # 添加新特征
    feature['my_new_feature'] = self._calc_my_feature(...)
    
    return feature
```

### Q18: 如何调整模型参数？
**A:** 在 `ZodiacPredictor.train()` 方法中修改：
```python
model = xgb.XGBClassifier(
    max_depth=8,          # 改这里
    learning_rate=0.1,    # 改这里
    n_estimators=300,     # 改这里
    ...
)
```

### Q19: 如何评估模型性能？
**A:** 
1. **历史回测**: 用过去的数据测试
2. **交叉验证**: TimeSeriesSplit 的结果
3. **实时监控**: 记录每期预测和实际结果

可以添加日志记录功能来追踪准确率。

### Q20: 可以导出模型供其他地方使用吗？
**A:** 可以。使用 pickle 保存：
```python
import pickle

# 保存
with open('model.pkl', 'wb') as f:
    pickle.dump(predictor.models, f)

# 加载
with open('model.pkl', 'rb') as f:
    models = pickle.load(f)
```

注意：需要同时保存 LabelEncoder 和特征工程配置。

## 📈 进阶问题

### Q24: 能否预测平码而不仅是特码？
**A:** 当前版本只预测特码生肖。预测平码需要：
1. 修改标签为多标签分类
2. 调整模型架构
3. 重新设计特征工程

这是一个更复杂的问题。

### Q25: 如何结合多个模型的预测结果？
**A:** 当前已经使用了3个 XGBoost 模型的平均。你可以：
1. 增加更多不同参数的模型
2. 使用加权平均（根据历史表现）
3. 使用堆叠（Stacking）方法

### Q26: 有没有移动端应用？
**A:** 目前没有。但你可以：
1. 在手机上安装 Python 环境
2. 使用 Termux（Android）或 Pyto（iOS）
3. 通过 SSH 远程运行脚本

未来可能开发 Web 界面或移动应用。
