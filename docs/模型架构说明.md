# 模型架构说明

## 概述

本系统采用 **XGBoost 集成学习** 架构，通过多个子模型的投票来提高预测的准确性和稳定性。

## 模型架构图

```
输入数据 (历史开奖记录)
    ↓
特征工程 (240+ 维特征)
    ↓
┌─────────────────────────┐
│   训练集 (80% 数据)      │
└─────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│     XGBoost 集成模型 (3个子模型)       │
│                                      │
│  Model 1 (seed=42)                   │
│  Model 2 (seed=123)                  │
│  Model 3 (seed=888)                  │
└──────────────────────────────────────┘
    ↓
概率平均融合
    ↓
遗漏值修正 + 热号微调
    ↓
输出：12个生肖的概率分布
```

## 核心组件

### 1. XGBoost 分类器

#### 模型参数
```python
{
    'objective': 'multi:softprob',  # 多分类软概率输出
    'num_class': 12,                 # 12个生肖
    'max_depth': 6,                  # 树的最大深度
    'learning_rate': 0.05,           # 学习率
    'n_estimators': 250,             # 树的数量
    'subsample': 0.8,                # 样本采样率
    'colsample_bytree': 0.8,         # 特征采样率
    'eval_metric': 'mlogloss'        # 评估指标
}
```

#### 参数说明
- **max_depth=6**: 控制模型复杂度，防止过拟合
- **learning_rate=0.05**: 较小的学习率，需要更多迭代但更稳定
- **n_estimators=250**: 足够的树数量保证学习效果
- **subsample=0.8**: 每次使用80%的样本，增加随机性
- **colsample_bytree=0.8**: 每次使用80%的特征，增加多样性

### 2. 集成策略

#### 为什么使用集成？
1. **降低方差**: 多个模型的平均可以减少单一模型的偶然性
2. **提高鲁棒性**: 不同随机种子捕捉不同的数据模式
3. **更好的泛化**: 集成模型通常在测试集上表现更稳定

#### 融合方法
```python
# 简单平均
final_proba = (proba_1 + proba_2 + proba_3) / 3
```

### 3. 类别权重平衡

#### 问题
某些生肖出现频率较高，模型可能偏向预测这些生肖。

#### 解决方案
根据各类别的频率动态调整权重：

```python
# 频率高的类别权重低，频率低的类别权重高
scale_pos_weight = {
    cls: 0.5 + 1.5 * (freq / max_freq) 
    for cls, freq in class_frequencies.items()
}
```

**效果：**
- 热门生肖（如羊）：权重约 2.0
- 冷门生肖（如狗）：权重约 0.9
- 平衡了模型对各类别的关注度

### 4. 时间序列交叉验证

#### 传统 CV 的问题
随机划分会破坏时间序列的顺序性，导致"未来信息泄露"。

#### 我们的方案
使用 `TimeSeriesSplit`：

```
Fold 1: [Train: 1-100] [Val: 101-150]
Fold 2: [Train: 1-150] [Val: 151-200]
Fold 3: [Train: 1-200] [Val: 201-250]
```

**优势：**
- 保持时间顺序
- 模拟真实预测场景
- 更可靠的性能评估

### 5. 后处理优化

#### 5.1 遗漏值修正
对于接近历史最大遗漏值的生肖，给予额外概率提升：

```python
if current_missing >= 0.9 * max_missing:
    boost = 0.02 * (current_missing / max_missing)
    result[zodiac] += boost
```

**原理：** 长期未出现的生肖有"回补"的可能性

#### 5.2 热号微调
近期频繁出现的生肖适度提升概率：

```python
hot_ratio = recent_count / max_count
result[zodiac] *= (1 + 0.03 * hot_ratio)
```

**原理：** 热号可能有持续性

#### 5.3 概率归一化
确保所有生肖概率之和为 1：

```python
total = sum(probabilities.values())
for zodiac in probabilities:
    probabilities[zodiac] /= total
```

## 训练流程

### Step 1: 数据准备
```python
# 加载历史数据
df = load_data()

# 构建特征
X, y = feature_engineer.build_features(df)

# 编码标签
y_encoded = label_encoder.fit_transform(y)
```

### Step 2: 数据集划分
```python
# 使用前80%作为训练集
split = int(len(X) * 0.8)
X_train, y_train = X[:split], y[:split]
```

### Step 3: 计算类别权重
```python
# 基于训练集统计
class_weights = compute_class_weights(y_train)
sample_weights = [class_weights[yi] for yi in y_train]
```

### Step 4: 训练多个模型
```python
seeds = [42, 123, 888]
for seed in seeds:
    model = XGBClassifier(..., random_state=seed)
    
    # 时间序列交叉验证
    for train_idx, val_idx in TimeSeriesSplit().split(X_train):
        model.fit(
            X_train[train_idx], 
            y_train[train_idx],
            sample_weight=sample_weights[train_idx],
            eval_set=[(X_train[val_idx], y_train[val_idx])]
        )
    
    # 最终在全量训练集上训练
    model.fit(X_train, y_train, sample_weight=sample_weights)
    
    models.append(model)
```

### Step 5: 预测与融合
```python
# 构建最新一期的特征
X_latest = build_latest_features(all_data)

# 每个模型预测
all_proba = [model.predict_proba(X_latest) for model in models]

# 平均融合
final_proba = np.mean(all_proba, axis=0)

# 后处理优化
final_proba = apply_corrections(final_proba, all_data)
```

## 模型评估

### 评估指标
- **准确率 (Accuracy)**: 预测正确的比例
- **对数损失 (Log Loss)**: 概率预测的质量
- **Top-K 准确率**: 前K个预测中包含真实值的比例

### 典型性能
基于历史回测：
- Top-1 准确率: ~15-20% (12分类，随机猜测为8.3%)
- Top-3 准确率: ~40-50%
- Top-5 准确率: ~60-70%

## 模型更新策略

### 何时重新训练？
1. **定期更新**: 每周或每期开奖后
2. **数据积累**: 新增一定数量的历史数据
3. **性能下降**: 近期预测准确率明显降低

### 增量学习
XGBoost 不支持真正的增量学习，需要：
1. 保存完整的历史数据
2. 重新训练整个模型
3. 替换旧模型

## 注意事项

1. **过拟合风险**: 彩票本质上是随机的，模型可能过拟合历史噪声
2. **数据质量**: 确保历史数据准确完整
3. **特征泄露**: 严格区分训练数据和测试数据的时间界限
4. **期望管理**: 模型只能提供概率参考，不能保证准确性

## 未来改进方向

1. **深度学习**: 尝试 LSTM、Transformer 等时序模型
2. **更多特征**: 加入外部因素（节假日、季节等）

4. **在线学习**: 实现实时更新机制
